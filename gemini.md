# Minecraft Texture Generator

This project automates the process of turning AI-generated (Nano Banana) images into perfect, scalable, Minecraft-ready sprite textures.

## How It Works

The core idea is to generate pixel art textures using AI, but with a **solid background color** (preferably a significantly darker shade of your chosen rarity color, e.g. dark purple background for an epic border). If rarities are desired, the AI prompt should explicitly request a "thick and identifiable [rarity color] border" and explicitly forbid a frame or border around the exterior canvas edge. The system then automatically detects the background color dynamically from the top-left pixel, strips it out entirely, crops the item to its precise boundaries, and rigorously rescales the image (using nearest-neighbor scaling) to specific Minecraft engine resolutions (16x16, 32x32, 64x64, 128x128, 256x256, 512x512).

Crucially, the system ensures that **every pixel in the end result is strictly 100% opaque or 100% transparent**. Minecraft does not support partial transparency on item textures and any semi-transparent pixels will cause severe rendering issues. 

## Project Structure

1. **`prompts.json`**: This JSON acts as the master manifest. For each item you want to generate, you define:
   * `"id"`: The unique identifier/name (e.g., `sapphire_sword`). This dictates the filename.
   * `"prompt"`: The exact prompt to give the AI image generator. Important prompt rules:
     * Request "pixel art" and "2D gaming asset".
     * Request a "totally solid [color] background" (e.g., pure dark purple) to ensure the main rarity-colored outline is not accidentally stripped if the item itself shares colors with the background.
     * Explicitly ban grids, lighting effects, or canvas-edge borders to ensure it calculates easily and crops cleanly.
     * Include a request for a thick rarity color border (e.g., epic purple) if leveraging the rarities system.
   * `"size"`: The strict square resolution the final image should be baked down to (e.g., 16, 32, 256).
   * `"base_rarity"` (Optional): Identifies the explicit border color generated by the AI image (defaults to `epic`).
   * `"rarities"` (Optional): An array of strings defining rarities to generate by substituting the `base_rarity` border color. Available rarities: `common`, `uncommon`, `rare`, `epic`, `legendary`, `relic`.
   * `"skins"` (Optional): An array of skin objects, each defining a `"prompt"`, `"id"`, and optionally `"rarities"`. Generating a skin with ID `[skin_id]` for an item with ID `[item_id]` expects a raw file named `[item_id]_[skin_id].png` (e.g., `flashbang_cheetah.png`) and automatically compiles its outputs into `output/[item_id]/skins/[skin_id]/`.

2. **`raw/`**: After you generate your textures with the solid backgrounds, save the unedited RAW images into this folder using the matching `"id"` from the JSON (e.g., `emerald_shield.png`).

3. **`output/`**: This is where your completed, cleanly cropped, perfectly scaled, binary-transparency `.png` sprites are exported. If you specify `rarities`, a folder will be dynamically created for that item instead, populated with individual files per rarity (e.g., `output/camera/epic.png`).

4. **`process_textures.py`**: The workhorse script. 

## The `process_textures.py` Pipeline Layer by Layer
* **Load manifest**: Opens `prompts.json` and gets the ID and requested scale for every listed item.
* **Locate raw**: Searches the `raw/` directory for `<id>.png`, `.jpg`, or `.jpeg`.
* **Detect Background dynamically**: Looks precisely at the **Top-Left** pixel (coordinate `0,0`). Whatever color that pixel is, the script assumes as the background key color for the rest of the image. This means you can use `#00FF00` (green), `#FF00FF` (magenta), `#FF0000` (red), etc., based on what contrasts best with the item.
* **Extract & Enforce strict Binary Alpha**: 
  * Any pixel matching the detected background color gets completely deleted—both RGB and Alpha zeroed out.
  * Any pixel *not* matching the detected background color gets its Alpha forcefully cranked to 255 (100% solid opacity). This fulfills the strict Minecraft engine requirement and entirely eliminates invisible/transparent anti-aliasing ghosting.
* **Halo Defringing (Borderless Mode)**:
  * For completely borderless items, the remaining pixels sitting directly on the anti-aliased edge contain a tint of the background color (e.g., a magenta halo). The script mathematically zeroes out the contribution of the background color specifically from the RGB values of these fringe pixels, turning them into a clean and sharp black/neutral stroke outline instead of a glowing magenta halo.
* **Recolor Rarities (Optional)**:
  * If the manifest includes `rarities`, the script analyzes the leftover anti-aliasing color outline (by finding the fractional background color component using linear algebra) and recolors the remaining border tint into defined structural rarity colors.
* **Crop & Rescale**: 
  * The script calculates a tight bounding box around the remaining solid pixels.
  * It then resizes that bounding box into the exact `"size"` variable defined in the JSON.
  * Because it is pixel art, it uses `Image.Resampling.NEAREST` mapping so edges remain perfectly sharp without interpolation blur.
* **Export**: The tightly packed item is pasted dead-center inside a transparent square, exactly the size specified, and saved to `output/`.

## Setup
Simply run `pip install -r requirements.txt` to install `Pillow` and `numpy`. 
To run a batch processor, save your raw images into `raw/` and execute `python process_textures.py`.
